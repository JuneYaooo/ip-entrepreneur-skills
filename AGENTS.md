# AGENTS.md -- 给 codex / aider / cursor 等 agent 的入口说明

本仓库是一个 IP 创业诊断与咨询 Skill，**权威文档在 [`SKILL.md`](./SKILL.md)**。任何涉及"分析账号/诊断/起号/定位/内容策略/变现/直播"的请求，都先完整读 `SKILL.md` 再动手，不要凭本文件的摘要就开跑——下面只是索引。

## 一分钟索引

- **核心方法论**：六维度框架——定位、人设、内容、数据、变现、长期（详见 `SKILL.md` 主体）
- **参考知识库**：`references/` 目录下 6 份深度资料（定位/内容/数据/直播/变现/增长），诊断时按需查阅
- **三种工作模式**：
  - 模式 A（IP 诊断）：分析现有账号的六维度健康度
  - 模式 B（从 0 到 1）：引导新账号完成定位、人设、对标、选题
  - 模式 C（增长突破）：诊断瓶颈，优化数据漏斗和变现路径
- **双输出版本**：每次都同时输出精华版（2-3 页，5 分钟读完）和全面版（六维度详析）
- **输出路径**：精华版 → `output/ip_brief_<对象名>.md`，全面版 → `output/ip_diagnosis_<对象名>.md`

## 调用规范（对 agent 的硬约束）

1. **先了解需求**：用户是诊断现有账号、从 0 起步还是突破瓶颈？
2. **按工作模式走**：每种模式有标准的分析维度和输出结构，详见 `SKILL.md`「工作模式」节
3. **参考知识库**：六维度深度方法论在 `references/` 目录，诊断时按对应维度查阅
4. **双输出**：精华版 + 全面版一起交付，不要只输出一个版本
5. **精华版为王**：即使只生成全面版，也要先让用户看到精华版的快速行动清单

## 不要做的事

- 不要凭空编造方法论——一切以 `SKILL.md` 和 `references/` 中的内容为准
- 不要只输出一个版本——精华版和全面版同时输出
- 不要在未分析完六维度前就给出结论
